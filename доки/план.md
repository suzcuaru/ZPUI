### ЭТАП 1: SQLite Database Layer
**Цель:** Надёжное хранение с историей и ротацией.

**Новые файлы:**
- `internal/database/db.go` — подключение, WAL, pragmas, graceful close.
- `internal/database/migrations/*.sql` — версионируемые миграции.
- `internal/database/models.go` — структуры с SQL-тегами.
- `internal/database/queries.go` — CRUD с context, prepared statements.
- `internal/database/action_log.go` — логирование действий.

**Схема БД (доработанная):**
```sql
-- Сессии приложения (не очищаются!)
CREATE TABLE sessions (
    id TEXT PRIMARY KEY,
    started_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    ended_at DATETIME,
    total_devices INTEGER DEFAULT 0
);

-- Устройства (привязаны к сессии)
CREATE TABLE session_devices (
    id TEXT PRIMARY KEY,
    session_id TEXT REFERENCES sessions(id),
    mac TEXT NOT NULL,
    ip TEXT,
    hostname TEXT,
    vendor TEXT, -- OUI lookup
    first_seen DATETIME,
    last_seen DATETIME,
    total_dl BIGINT DEFAULT 0,
    total_ul BIGINT DEFAULT 0,
    is_online BOOLEAN DEFAULT FALSE
);

-- Соединения (ротируются: 7 дней)
CREATE TABLE device_connections (
    id TEXT PRIMARY KEY,
    device_id TEXT REFERENCES session_devices(id),
    dst_host TEXT,
    dst_port INTEGER,
    bytes_dl BIGINT,
    bytes_ul BIGINT,
    started_at DATETIME,
    closed_at DATETIME
);

-- Логи действий (ротируются: 90 дней)
CREATE TABLE action_logs (
    id TEXT PRIMARY KEY,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    level TEXT CHECK(level IN ('debug','info','warn','error','fatal')),
    category TEXT,
    action TEXT,
    details JSON,
    pii_masked BOOLEAN DEFAULT TRUE
);

-- Снапшоты трафика (ротируются: 30 дней)
CREATE TABLE traffic_snapshots (
    id TEXT PRIMARY KEY,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    dl_speed REAL,
    ul_speed REAL,
    total_dl BIGINT,
    total_ul BIGINT,
    conn_count INTEGER
);

-- Индексы
CREATE INDEX idx_dev_mac ON session_devices(mac);
CREATE INDEX idx_conn_device ON device_connections(device_id);
CREATE INDEX idx_log_ts ON action_logs(timestamp);
CREATE INDEX idx_snap_ts ON traffic_snapshots(timestamp);
```

**Ротация:** Background worker каждые 24ч удаляет старые записи согласно политике.

**Definition of Done:**
- [ ] БД создаётся автоматически, миграции применяются.
- [ ] WAL mode включён, `synchronous=NORMAL`.
- [ ] Graceful shutdown корректно закрывает DB.
- [ ] Unit-тесты на queries (coverage > 80%).
- [ ] Ротация работает без блокировок.

### ЭТАП 2: Device Tracking System
**Цель:** Точный трекинг устройств с live-обновлениями.

**Новые файлы:**
- `internal/devices/tracker.go` — агрегация, LRU cache, worker pool.
- `internal/devices/oui.go` — локальный OUI lookup.
- `internal/devices/detector.go` — OS-specific обнаружение (WMI/arp/sysctl).
- `internal/web/api_devices.go` — REST + SSE endpoints.

**API Endpoints:**
| Method | Path | Описание |
|--------|------|----------|
| GET | `/api/v1/devices?page=&limit=&status=` | Пагинированный список |
| GET | `/api/v1/devices/:id` | Детали + история |
| GET | `/api/v1/devices/:id/connections?page=` | Соединения устройства |
| POST | `/api/v1/devices/:id/ping` | Async ping (возвращает task_id) |
| GET | `/api/v1/devices/stream` | SSE live updates |
| DELETE | `/api/v1/devices/:id` | Забыть устройство |

**Ключевые улучшения:**
- **Fallback ID:** Если MAC отсутствует → hash(IP+port+hostname).
- **Rate Limiting:** Token bucket 60 req/min на ping.
- **SSE Reconnection:** Heartbeat 30s, exponential backoff на клиенте.
- **Virtualization-ready:** API возвращает только необходимые поля для карточек.

**Definition of Done:**
- [ ] 100+ устройств без деградации (<100ms response).
- [ ] SSE пересоединяется автоматически.
- [ ] Ping не блокирует основной поток.
- [ ] Integration тесты с реальной БД.

### ЭТАП 6: Frontend Logger & Telemetry
**Цель:** Надёжная телеметрия с zero data loss.

**Архитектура:**
```typescript
class Logger {
  private queue: LogEvent[] = [];
  private readonly MAX_BATCH = 50;
  private readonly FLUSH_INTERVAL = 5000;
  
  // Queue in IndexedDB for offline resilience
  async log(category: string, action: string, details?: object) {
    const event = this.createEvent(category, action, details);
    await this.enqueue(event);
    this.flushIfNeeded();
  }
  
  // SendBeacon on page unload
  private setupUnloadHandler() {
    window.addEventListener('beforeunload', () => {
      navigator.sendBeacon('/api/v1/logs/frontend', 
        JSON.stringify({ events: this.queue })
      );
    });
  }
  
  // Retry with exponential backoff
  private async flush() {
    try {
      await fetch('/api/v1/logs/frontend', { ... });
    } catch {
      this.scheduleRetry(); // 1s → 2s → 4s → 8s → max 30s
    }
  }
}
```

**Sampling:** Click events логируются с вероятностью 10% (configurable).

**Definition of Done:**
- [ ] Data loss < 0.1% при crash/close.
- [ ] Retry logic с backoff.
- [ ] Sampling работает.
- [ ] PII masked client-side before send.

## 4. РИСКИ И MITIGATION STRATEGIES

| Риск | Вероятность | Impact | Mitigation |
|------|-------------|--------|------------|
| SQLite locking under load | Medium | High | WAL + connection pool + read replicas |
| Data loss on crash | Medium | Critical | fsync + journaling + sendBeacon |
| MAC spoofing | High | Medium | Multi-factor ID (MAC+IP+hostname hash) |
| PII leakage via logs | Medium | Critical | Middleware masking + client-side mask |
| SSE connection leaks | High | Medium | Heartbeat 30s + timeout + cleanup |
| Memory leak in tracker | Medium | High | pprof + LRU + memory limits |
| XSS via user input | Medium | Critical | CSP + sanitization + React defaults |
| Breaking API changes | Medium | High | `/api/v1/` versioning |
| Legacy Windows compat | Medium | Medium | Test matrix Win7-11 + graceful fallbacks |
| CGO cross-compile issues | Low | Medium | modernc.org/sqlite (pure Go) |
