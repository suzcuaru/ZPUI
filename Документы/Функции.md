# Функции ZPUI

> Документ описывает **пользовательские возможности** продукта — что конкретно может делать пользователь в интерфейсе, какие задачи решает, какие элементы UI для этого существуют. Технические детали — в [`Архитектура.md`](./Архитектура.md), доменные правила — в [`Бизнес-логика.md`](./Бизнес-логика.md), визуальный язык — в [`Дизайн-и-типографика.md`](./Дизайн-и-типографика.md).

## 2. Дашборд (`/`)

Главный экран — оперативная сводка состояния.

### 2.1. Элементы

- **Большая карточка статуса движка:** «Обход: Включён / Выключен / Ошибка» с цветовым кодированием (зелёный/серый/красный). Кнопка «Старт» / «Стоп» прямо на карточке.
- **Кольцевой индикатор доступности:** 0–100% за последнее измерение. Цветовая шкала: ≥80% зелёный, 50–80% жёлтый, <50% красный.
- **Активная стратегия:** имя карточки стратегии с возможностью перейти к смене.
- **Мини-график за час:** спарклайн доступности.
- **Состояние сервисов:** компактные бейджи «Прокси: вкл/выкл», «DNS: вкл/выкл».
- **Последнее событие:** строка из журнала (например, «AutoSwitch переключился на … 5 мин назад»).
- **Кнопка «Диагностика»:** быстрый доступ при проблемах.

### 2.2. Быстрые действия

- «Старт/Стоп» — основное действие.
- «Автовыбор» — запуск автоподбора (с подтверждением, т. к. процесс длительный).
- «Открыть Discord» — кнопка быстрого теста (открывает браузер на discord.com).

## 3. Стратегии (`/strategies`)

Управление пресетами zapret.

### 3.1. Список стратегий

- Карточный вид, каждая карточка содержит:
  - имя (и технический id);
  - краткое описание (для какой блокировки/DPI);
  - бейдж «Активна» (если выбрана);
  - бейдж «Рекомендуется» (если autoselet ранее её выбрал);
  - кнопка «Применить».

### 3.2. Применение стратегии

- Клик «Применить» → движок перезапускается с новыми аргументами → `config.activeStrategy` обновляется.
- В UI — индикатор применения (спиннер), затем подтверждение.

### 3.3. Автоподбор

- Кнопка «Автовыбор» открывает модалку с прогрессом:
  - текущая тестируемая стратегия;
  - прогресс-бар (N из M);
  - результат каждой (доступность %);
  - кнопка «Отмена».
- По завершении — результат с кнопкой «Применить лучшую» (или автоприменение, если в настройках включено).

### 3.4. Детали стратегии

- Клик на карточку → модальное окно с:
  - полным описанием;
  - аргументами CLI (для гиков);
  - историей применения (из БД).

## 4. Списки доменов (`/lists`)

Редактор фильтр-списков zapret.

### 4.1. Разделы

- **Discord-список:** домены Discord.
- **YouTube-список:** домены YT + googlevideo.
- **Пользовательский список:** домены, добавленные пользователем.
- **Исключения:** домены, которые НЕ нужно трогать.

### 4.2. Операции

- Просмотр (табличный вид).
- Добавление домена (поле + кнопка; валидация формата).
- Удаление (крестик в строке).
- Редактирование (инлайн).
- Импорт из текста (вставить список → распарсить по строкам).
- Экспорт в `.txt`.
- Сброс к дефолту.

### 4.3. Сохранение

- Изменения применяются при следующем старте движка.
- `useDebouncedSave` — автосохранение через 1 сек после правки.
- Индикатор «Не сохранено» / «Сохранено».

## 5. Прокси (`/proxy`)

Управление SOCKS5-сервером для раздачи обхода.

### 5.1. Тумблер и статус

- Крупный `Switch`: Включён / Выключен.
- Статус: «Слушает на 192.168.1.10:1080» или «Остановлен».

### 5.2. Настройки

- Порт (по умолчанию 1080).
- Авторизация (вкл/выкл + поля login/password с маской).
- Привязка к интерфейсу (все / конкретный IP).

### 5.3. Подключение клиентов

- Отображение адреса `socks5://192.168.1.10:1080` (с кнопкой «Копировать»).
- **QR-код** для быстрого сканирования телефоном.
- Краткая инструкция «Как настроить на телефоне» (с раскрывающимися блоками для iOS/Android).

### 5.4. Мониторинг подключений

- Список активных клиентов (IP, длительность, объём трафика).
- Опционально — лог подключений.

## 6. Xbox DNS (`/dns`)

Управление внутренним DNS-сервером для игровых консолей.

### 6.1. Тумблер и статус

- `Switch`: Включён / Выключен.
- Статус: «Слушает на 53/UDP» или «Остановлен».
- Если порт занят — понятная ошибка с инструкцией.

### 6.2. Настройки

- Режим: «Только Xbox» / «Полный резолвер».
- Список доменов Xbox (предзаполнен, редактируем).
- Upstream DNS (для проксирования не-Xbox запросов).

### 6.3. Подключение консоли

- Отображение IP компьютера как DNS-сервера.
- Инструкция для Xbox / PlayStation с пошаговым описанием настроек сети.

---

## 7. Монитор (`/monitor`)

История и графики доступности.

### 7.1. Фильтры периода

- Час / День / Неделя.

### 7.2. Визуализации

- **Линейный график доступности** за выбранный период (ось X — время, ось Y — %).
- **График задержки** (latency) по ресурсам.
- **Таблица ресурсов:** имя, средняя доступность за период, текущий статус, количество сэмплов.
- **Топ проблемных:** ресурсы с наибольшей долей неудач.

### 7.3. Экспорт

- Экспорт истории в CSV (для анализа пользователем).

### 7.4. Настройка пула

- Кнопка «Редактировать ресурсы» → модалка со списком доменов (добавить/удалить).

## 8. Диагностика (`/diagnostics`)

Самопроверка системы и рекомендации.

### 8.1. Список проверок

- Карточный список всех проверок (см. [`Бизнес-логика.md`](./Бизнес-логика.md) §10):
  - статус (зелёный/жёлтый/красный);
  - краткое описание;
  - рекомендация (с кнопкой действия, если применимо).

### 8.2. Действия из диагностики

- «Перезапустить движок».
- «Переустановить zapret».
- «Открыть исключения антивируса» (ссылка на инструкцию).
- «Запустить автоподбор».
- «Сбросить настройки к дефолту».

### 8.3. Логи

- Вкладка «Логи» внутри диагностики.
- Просмотр `logs/*.log` с фильтром по уровню (INFO/WARN/ERROR).
- Кнопка «Копировать» (для отправки в баг-репорт).
- Кнопка «Открыть папку логов» (в Explorer).

### 8.4. Сообщить о баге

- Кнопка «Сообщить о баге» → форма с автозаполнением версии, ОС, диагностического снапшота → ссылка на GitHub Issues.

## 9. Обновления (`/updates`)

Проверка и применение обновлений.

### 9.1. Текущие версии

- Карточка с версиями:
  - ZPUI core (из `version.txt`).
  - 4 модулей (wizard, autoselect, selfupdate, zapretupdate — из `versions.json`).
  - zapret (из `zapret/version.txt`).

### 9.2. Проверка

- Кнопка «Проверить обновления».
- Результат: «У вас актуальные версии» ИЛИ «Доступно обновление zapret vX.Y.Z».
- Опция «Проверять автоматически при старте».

### 9.3. Применение

- Для zapret: кнопка «Обновить» → прогресс → резервная копия → применение → перезапуск движка.
- Для ZPUI: кнопка «Обновить» → скачивание → запуск selfupdate → подтверждение перезапуска.

### 9.4. История обновлений

- Список прошлых применённых обновлений (из БД событий).

## 10. Настройки (`/settings`)

Все пользовательские предпочтения.

### 10.1. Общие

- **Язык интерфейса:** русский / английский (мгновенное переключение).
- **Тема:** тёмная / светлая / системная.
- **Акцентный цвет:** палитра пресетов + кастомный color picker.
- **Автозапуск с Windows:** тумблер (пишет в реестр).
- **Запуск свёрнутым в трей:** тумблер.

### 10.2. Движок

- **Автозапуск zapret при старте ZPUI:** тумблер.
- **Поведение при ошибке:** «Молча рестарт» / «Показывать тост» / «Открывать диагностику».
- **Лимит рестартов:** слайдер (1–10 за 60 сек).

### 10.3. AutoSwitch

- **Включить:** тумблер.
- **Порог доступности:** слайдер (0–100%).
- **Cooldown:** селектор (5m / 10m / 30m / 1h).
- **Чувствительность (consecutiveFails):** слайдер (1–5).
- **Пул стратегий:** мультиселект карточек.

### 10.4. Мониторинг

- **Интервал поллинга:** селектор (15s / 30s / 1m / 5m).
- **Пул ресурсов:** редактор списка доменов.
- **Хранение истории:** селектор (7 / 14 / 30 дней).

### 10.5. Уведомления

- **Тосты:** вкл/выкл.
- **Balloon в трее:** вкл/выкл.
- **Звук при критических событиях:** вкл/выкл.
- **Типы событий:** чекбоксы (engine start/stop, autoswitch, updates, ...).

### 10.6. Прокси

- (дублирует `/proxy` настройки, в виде компактного блока).

### 10.7. DNS

- (дублирует `/dns`).

### 10.8. Сброс

- «Сбросить настройки к дефолту» (с подтверждением).
- «Сбросить wizard» (принудительно запустить мастер при следующем старте).

## 11. Мастер первого запуска (Wizard, оверлей)

Пошаговый мастер установки и настройки.

### 11.1. Шаги

1. **Приветствие:** краткое описание продукта, кнопка «Начать».
2. **Проверка системы:** автоматический чек (ОС, разрядность, права, антивирус) с отображением результата каждого пункта.
3. **Установка zapret:** прогресс скачивания + распаковки + проверки целостности.
4. **Автоподбор:** прогресс перебора стратегий.
5. **Готово:** сводка — выбранная стратегия, доступность, кнопка «Открыть дашборд».

### 11.2. Навигация

- Кнопки «Назад» / «Далее» (где применимо).
- Кнопка «Пропустить» (для шагов, которые можно отложить — например, автоподбор).
- Прогресс-бар сверху.
- Запрет закрытия без подтверждения на критических шагах.

## 12. Трей и уведомления

### 12.1. Иконка трея

- Отображается всегда при запущенном ZPUI.
- Левый клик — показать/скрыть окно.
- Правый клик — контекстное меню.

### 12.2. Контекстное меню

- «Показать ZPUI».
- «Старт/Стоп обхода».
- «Автовыбор».
- «Разделитель».
- «Прокси: вкл/выкл».
- «DNS: вкл/выкл».
- «Разделитель».
- «Выход» (корректное завершение, не просто скрытие).

### 12.3. Balloon-уведомления

- Запуск движка.
- AutoSwitch сработал.
- Доступно обновление.
- Критическая ошибка (движник упал и не поднялся).

### 12.4. Поведение кнопки «Закрыть»

- По умолчанию: закрытие окна = сворачивание в трей (приложение продолжает работать).
- Опция в настройках: «При закрытии сворачивать в трей» / «Полностью завершать».

## 13. Глобальное поведение

### 13.1. Окно

- Фиксированный размер **960×640** (нельзя развернуть, нельзя изменить размер — см. `window_windows.go`).
- Отключена кнопка maximize.
- Позиция — запоминается между запусками.

### 13.2. Тосты (in-app)

- Всплывающие уведомления внутри окна.
- Типы: success / info / warning / error.
- С кнопками действия («Открыть», «Отменить», ...).
- Автоисчезание через N секунд (настраивается).

### 13.3. Модальные окна

- Подтверждения критичных действий (сброс, обновление, выход при активном движке).
- Формы ввода.

### 13.4. Горячие клавиши (опционально)

- `Ctrl+,` — настройки.
- `Ctrl+D` — дашборд.
- `Esc` — закрыть модалку.

### 13.5. Контекстное меню текста

- Стандартное (копировать/вставить) в полях ввода.

## 14. Доступность и UX-принципы

- **Минимум кликов** для частых действий (старт/стоп — одна кнопка на дашборде).
- **Всегда видимый статус** движка (даже в других вкладках — через бейдж в шапке).
- **Понятные сообщения об ошибках** — без стектрейсов, с рекомендацией.
- **Подтверждения для деструктивных действий** (сброс, удаление списков).
- **Локализация** всех строк (см. [`Дизайн-и-типографика.md`](./Дизайн-и-типографика.md) §7).
- **Тёмная тема по умолчанию** (привычная для целевой аудитории).
- **Поддержка DPI-scaling Windows** (читаемость на HiDPI-экранах).
