# Концепция продукта ZPUI

> Документ описывает продуктовое видение, проблемную область, целевую аудиторию, ценностное предложение и принципы проектирования ZPUI. Это верхнеуровневый документ: технические детали реализации — в [`Архитектура.md`](./Архитектура.md), поведенческие сценарии — в [`Бизнес-логика.md`](./Бизнес-логика.md), перечень возможностей — в [`Функции.md`](./Функции.md), визуальный язык — в [`Дизайн-и-типографика.md`](./Дизайн-и-типографика.md).

---

## 1. Одно предложение

**ZPUI — это настольное приложение для Windows, которое одним нажатием включает обход блокировок Discord, YouTube и других ресурсов с помощью движка [zapret](https://github.com/bol-van/zapret), и дополнительно умеет раздавать этот обход на другие устройства в домашней сети (через SOCKS-прокси и DNS для игровых консолей).**

---

## 2. Проблемная область

### 2.1. Что такое DPI и почему сайты «не открываются»

**DPI (Deep Packet Inspection)** — технология анализа сетевого трафика на стороне провайдера. В России (и ряде других стран) DPI используется операторами связи для блокировки доступа к ресурсам по решению регулятора.

Технически блокировка работает так:

1. Пользователь открывает, например, `discord.com` или `youtube.com`.
2. Провайдер перехватывает пакеты (TLS ClientHello, SNI), видит целевой домен.
3. Провайдер разрывает соединение (TCP RST, подменённый TLS Alert, сброс NAT) либо «глушит» трафик.

Результат: сайт «не грузится», приложение не подключается, видео тормозит.

### 2.2. Что такое zapret

**zapret** — open-source утилита (автор `bol-van`), которая обходит DPI на уровне локальной машины. Принципы:

- Перехват трафика через драйвер ядра **WinDivert** (Windows).
- Модификация пакетов так, чтобы сигнатура, по которой DPI опознаёт целевой домен, искажалась: фрагментация TLS ClientHello, подмена порядка сегментов, манипуляции с TTL, fake-пакеты, обнуление TCP-флагов и т. п.
- DPI «не узнаёт» домен и пропускает трафик — сайт открывается.

Конкретный дистрибутив, на который опирается ZPUI, — **[Flowseal/zapret-discord-youtube](https://github.com/Flowseal/zapret-discord-youtube)**: готовая сборка zapret с набором пресетов-стратегий (`.bat`-файлы с аргументами для `winws.exe`), заточенная под Discord и YouTube.

### 2.3. В чём сложность для конечного пользователя

zapret — мощный, но «недружелюбный» инструмент:

- **CLI-only**: настройка через `.bat`-файлы, параметры командной строки `winws.exe`.
- **Десятки стратегий**: под каждого провайдера/DPI нужна своя — угадать сложно, нужно тестировать.
- **Требуются права администратора** (драйвер WinDivert работает в ring 0).
- **Антивирусы ругаются** на драйвер перехвата трафика (ложное срабатывание).
- **Не понятно, работает или нет**: zapret молчит, ошибку нужно искать в логах.
- **Нет мониторинга**: не видно, какой объём трафика идёт через обход, какие устройства подключены.
- **Нельзя раздать на телефон/консоль**: zapret работает только на одной машине.
- **Обновления ручные**: вышла новая стратегия под изменившийся DPI — пользователь об этом не узнает.

### 2.4. Масштаб проблемы

Блокировки затрагивают десятки миллионов пользователей в РФ. Discord (мессенджер №1 у геймеров и ИТ-сообществ), YouTube (крупнейший видеохостинг), а также периодически Telegram, Twitch, Instagram, зарубежные СМИ. Спрос на простой инструмент обхода — массовый и постоянный: при каждом усилении блокировок аудитория растет.

---

## 3. Решение

ZPUI — это **«графическая оболочка» над zapret + набор связанных сервисов**, упакованная в привычный Windows-инсталлятор. Принцип: скачал, поставил, запустил — и через 3 минуты Discord работает.

### 3.1. Что именно делает ZPUI

| Задача пользователя | Без ZPUI | С ZPUI |
|---|---|---|
| Установить zapret | Скачать архив, распаковать, разобраться со структурой папок | Нажать «Установить» — скачается и распакуется автоматически |
| Подобрать рабочую стратегию | По очереди запускать `.bat`, проверять сайт вручную | Нажать «Автовыбор» — ZPUI сам переберёт стратегии, проверит доступность и выберет лучшую |
| Понять, работает ли обход | Читать консоль или смотреть в логи | Видеть процент доступности ресурсов на дашборде в реальном времени |
| Раздать обход на телефон | Ставить отдельный прокси-сервер, настраивать руками | Включить SOCKS5-прокси в UI, ввести IP:порт на телефоне |
| Раздать обход на Xbox/PlayStation | Невозможно или крайне сложно | Включить DNS, указать IP компьютера как DNS-сервер на консоли |
| Обновиться при смене DPI | Следить за репозиторием, переустанавливать | ZPUI сам проверяет обновления zapret, скачивает и применяет |
| Диагностировать «ничего не работает» | Гуглить, читать форумы | Кнопка «Диагностика» проверяет систему и даёт рекомендации |

### 3.2. Принципы проектирования продукта

1. **Один клик до результата.** Минимум шагов от запуска до работающего Discord. Первый запуск — мастер, который сам всё ставит и настраивает.
2. **Прозрачность состояния.** Пользователь в любой момент видит: запущен ли обход, какой процент ресурсов доступен, какая стратегия активна, какой трафик идёт. Никаких «чёрных ящиков».
3. **Дружелюбие к не-ИТ-пользователю.** Никакого жаргона в основном UI. Стратегии — карточки с понятными названиями, ошибки — тосты с кнопкой «Сообщить о баге» и копированием текста.
4. **Не мешать.** Окно фиксированного компактного размера 960×640. Сворачивается в трей, работает в фоне, не показывает лишних уведомлений (всё настраиваемо).
5. **Локализация на русский.** Россия — основной рынок. Русский язык по умолчанию, английский — опционально.
6. **Самодостаточность.** Не требует установки Python, Node.js, драйверов вручную. Не требует аккаунта. Не требует интернет-соединения для управления (только для скачивания zapret и обновлений).
7. **Безопасность и доверие.** Проверка целостности файлов (SHA-256 манифест с Yandex Disk), опциональная проверка на VirusTotal. Прозрачные обновления с резервным копированием.
8. **Кросс-разрядность.** Работает и на 32-битной, и на 64-битной Windows 10/11.

## 4. Целевая аудитория

### 4.1. Основной сегмент: «обычный пользователь»

- Возраст 14–45, активный пользователь Discord/YouTube.
- ИТ-грамотность базовая: умеет скачать и запустить `.exe`, но не умеет/не хочет возиться с командной строкой.
- География: преимущественно Россия, частично СНГ.
- Мотивация: «хочу, чтобы Discord/YouTube просто работали».
- Устройство: один домашний ПК/ноутбук с Windows.

### 4.2. Расширенный сегмент: «раздать на все устройства»

- Семья с несколькими устройствами: ПК + телефоны + планшеты + Smart TV + игровая консоль.
- Мотивация: один компьютер как «шлюз» с обходом, остальные устройства подключаются к нему через прокси/DNS.
- Это и есть сценарий SOCKS5-прокси и Xbox DNS.

### 4.3. Технический сегмент: «гик / администратор»

- ИТ-специалист, который хочет тонкой настройки: выбор конкретной стратегии, ручное редактирование списков доменов, game-filter для онлайн-игр, IPSet-фильтры, мониторинг трафика.
- Мотивация: контроль и понимание, что происходит «под капотом».
- ZPUI удовлетворяет этот сегмент через вкладки «Стратегии», «Списки», «Монитор», «Диагностика», редактор логов.

### 4.4. Нецелевая аудитория

- Пользователи macOS / Linux (ZPUI только под Windows).
- Корпоративные сети с жёстким DPI и proactive defence (решается индивидуально, ZPUI не претендует).
- Те, кому нужен VPN-туннель «в другую страну» — ZPUI **не меняет** IP-адрес и **не шифрует** трафик глобально, он только чинит доступ к конкретным заблокированным ресурсам.

## 5. Ценностное предложение

| Аспект | Значение |
|---|---|
| **Время до результата** | ~3 минуты от запуска инсталлятора до работающего Discord (мастер первого запуска) |

| **Необходимые знания** | Уметь запускать `.exe` |

| **Поддержка архитектур** | Windows 10/11, x86 и x64 |

## 6. Ключевые сценарии использования

### Сценарий 1. «У меня перестал работать Discord»

1. Пользователь скачивает инсталлятор со страницы релизов.
2. Устанавливает как обычную программу (или запускает портативную версию).
3. При первом запуске мастер проверяет систему, скачивает zapret, автоподбирает стратегию.
4. Discord работает.

### Сценарий 2. «Хочу раздать обход на телефон»

1. На ПК открывает вкладку «Прокси», включает SOCKS5.
2. Видит адрес `socks5://192.168.1.10:1080` и QR-код.
3. На телефоне в настройках Wi-Fi вводит адрес прокси.
4. На телефоне открываются заблокированные сайты.

### Сценарий 3. «Хочу, чтобы Xbox получал обход»

1. На ПК открывает вкладку «DNS», включает внутренний DNS-сервер.
2. На Xbox в настройках сети указывает IP компьютера как DNS.
3. Xbox-сервисы (Store, multiplayer matching, media) начинают работать стабильно.

### Сценарий 4. «Прекратилось работать — что-то сломалось»

1. Открывает дашборд — видит, что доступность упала до 20%.
2. Заходит в «Запрет» → «Диагностика» — получает список проверок и рекомендаций.
3. Если стратегия «протухла» — нажимает «Автовыбор», ZPUI перебирает стратегии и переключается.
4. Либо нажимает «Переустановить» — служба пересоздаётся.
5. В крайнем случае — обновляет zapret до свежей версии (новые стратегии под новый DPI).

### Сценарий 5. «AutoSwitch — самоизлечение»

1. Пользователь включает в настройках режим AutoSwitch с порогом, например, 60%.
2. ZPUI фоном каждые N минут проверяет доступность ресурсов.
3. Если падает ниже порога — автоматически пробует другие стратегии из списка.
4. Нашёл рабочую — переключился, показал уведомление. Без участия пользователя.

## 7. Конкурентное окружение

### 7.1. Прямые аналоги по zapret

- **DiscordYouTubeFix** и подобные `.bat`-сборки — CLI/пакетные, без UI, без мониторинга, без прокси.
- **GoodbyeDPI** (ValdikSS) — другой движок (подменяет SNI на уровне WinDivert), тоже CLI-only; есть сторонние графические обёртки, но без экосистемы (прокси, DNS, автообновление, мониторинг).

### 7.2. VPN-сервисы

- **Платные VPN** (ExpressVPN, Nord, Surfshark, российские) — шифруют весь трафик, меняют геолокацию. Дороже, могут сами блокироваться, бьют по скорости всего интернета. ZPUI не конкурирует напрямую: он точечно чинит доступ, не трогая остальной трафик.
- **Бесплатные VPN** — нестабильны, небезопасны, показывают рекламу.

### 7.3. Чем ZPUI отличается

| Критерий | zapret (CLI) | Сторонние GUI-обёртки | VPN | **ZPUI** |
|---|---|---|---|---|
| Простота для новичка | ✗ | ~ | ✓ | ✓ |
| Не трогает остальной трафик | ✓ | ✓ | ✗ | ✓ |
| Бесплатно | ✓ | ✓ | ✗ | ✓ |
| Раздача на другие устройства | ✗ | ✗ | зависит | ✓ (SOCKS5 + DNS) |
| Автоподбор стратегии | ✗ | редко | н/д | ✓ |
| Автообновление стратегий | ✗ | редко | н/д | ✓ |
| Мониторинг доступности | ✗ | ✗ | н/д | ✓ |
| Самодиагностика | ✗ | ✗ | н/д | ✓ |
| AutoSwitch (самоизлечение) | ✗ | ✗ | н/д | ✓ |

## 8. Бизнес-модель

**ZPUI — бесплатный open-source проект (лицензия MIT).**

- **Цена для пользователя: 0 ₽.** Навсегда.
- **Монетизации нет.** Никакой рекламы, подписок, телеметрии, paywalls, ограничений по функциям.
- **Финансирование:** проект поддерживается автором на энтузиазме; возможны донаты через GitHub Sponsors (на усмотрение автора).
- **Дистрибуция:**
  - **GitHub Releases** — основной канал (`github.com/suzcuaru/ZPUI/releases`).
  - **Яндекс.Диск** — резервное зеркало на случай блокировки GitHub (`disk.yandex.ru/d/...`), а также первичный канал обновления самого ZPUI (satellite-обновлятор качает zip с Диска).
  - Обновление zapret идёт с GitHub (`Flowseal/zapret-discord-youtube`).

### Почему бесплатно и открыто

1. **Социальная миссия:** доступ к информации и коммуникации — базовое право. Блокировки — попытка это право ограничить; инструмент обхода должен быть доступен всем, а не только тем, кто готов платить.
2. **Доверие:** open-source код + MIT означают, что любой может проверить, что ZPUI не делает ничего скрытого (не ворует данные, не майнит, не шпионит). Критично для инструмента, требующего прав администратора.
3. **Устойчивость к давлению:** открытый код нельзя «закрыть» давлением на одну компанию — форки поддерживают жизнь проекта.

## 9. Допущения, ограничения, риски

### 9.1. Допущения

- Пользователь работает под Windows 10 или 11 (32 или 64 бита).
- Пользователь имеет права администратора (требуется для драйвера WinDivert).
- У пользователя есть доступ в интернет хотя бы к одному из каналов: GitHub или Яндекс.Диск.
- Антивирус либо не установлен, либо папка ZPUI добавлена в исключения (иначе удаляется драйвер WinDivert).

### 9.2. Технические ограничения

- **Только Windows.** zapret использует WinDivert — драйвер только под Windows. macOS/Linux требуют другого движка.
- **Только IPv4-сценарии** (в типовом случае). Переход провайдеров на full-IPv4-over-something может потребовать новых стратегий.
- **Не работает против глубоких блокировок active probing** (когда провайдер сам устанавливает соединение и проверяет отклик сервера).
- **Не шифрует трафик глобально и не меняет IP.** Если задача — анонимность или доступ к geo-blocked контенту из другой страны, нужен VPN, а не zapret.
- **Окно фиксированного размера 960×640.** Это сознательное решение для компактного десктоп-приложения (не меняется в production; `window_windows.go` отключает кнопку maximize).

### 9.3. Продуктовые риски

| Риск | Влияние | Митигация |
|---|---|---|
| Провайдер меняет DPI — текущие стратегии перестают работать | Критичное: у всех пользователей «всё ломается» | (1) Автообновление zapret под новый DPI; (2) AutoSwitch; (3) автор публикует hotfix-релиз |
| GitHub блокируют в РФ | Невозможно скачать/обновиться | Резервное зеркало на Яндекс.Диске; satellite-обновлятор использует Диск как primary |
| Антивирус удаляет WinDivert | У части пользователей не запускается | Инструкция в README, проверка целостности файлов с подсветкой недостающих |
| Ложное срабатывание антивируса на сам ZPUI | Падение доверия | Проверка через VirusTotal (встроенный security-модуль), открытый код, code signing при возможности |
| Драйвер WinDivert конфликтует с другим ПО (антивирусы, фаерволы) | Нестабильность | Диагностика, документация по конфликтам |

## 10. Дорожная карта и эволюция

Текущее состояние (на момент документа): **зрелый продукт v1.4.x**, активно используется. Развитие идёт по направлениям:

1. **Адаптация к новым DPI.** Каждое изменение блокировок → новая стратегия в zapret → автообновление.
2. **Улучшение диагностики.** Более точное определение причины «не работает», автоподсказки.
3. **Расширение мониторинга.** Глубже: какие ресурсы, по каким слоям блокируются, история за день/неделю.
4. **Безопасность обновлений.** Подписанные манифесты, интеграция VirusTotal (модуль `internal/security/`).
5. **UX первого запуска.** Сокращение времени до результата, fewer clicks.
6. **Локализация.** Английский (уже есть), потенциально другие языки.

Поддержка других платформ (macOS/Linux) в дорожной карте **не планируется** — это потребовало бы переписать движок с WinDivert на альтернативы (nfqueue, netgraph).

## 11. Принципы именования и позиционирования

- **Название:** `ZPUI` — сокращение от «**Z**apret **P**oint **U**ser **I**nterface» (рабочее внутреннее имя; закрепилось как бренд).
- **Продуктовое имя в UI:** «ZPUI» (без расшифровки).
- **Движок:** «zapret» (в UI — «Запрет», поскольку русский основной язык).
- **Тон коммуникации:** спокойный, технически точный, без хайпа. Никаких обещаний «100% гарантия обхода» — DPI меняется, честно говорим, что инструмент может перестать работать, но оперативно обновляется.
