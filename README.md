# ZPUI

**Модульная оболочка (modular shell) для Windows.** Версия `0.0.0`.

ZPUI — это базовое приложение, которое само по себе почти ничего не делает: только интерфейс,
системный трей и собственную конфигурацию. Весь функционал добавляется **модулями** —
отдельными `.exe`-файлами со своими версиями и разработкой.

## Принцип

```
zpui.exe            ← ядро (этот проект): UI + трей + система модулей
modules/
  ├─ my-mod/
  │   ├─ module.json   ← манифест
  │   ├─ my-mod.exe    ← ядро модуля
  │   └─ ...           ← доп. файлы
  └─ another-mod/
      ├─ module.json
      └─ ...
```

- Ядро работает **само по себе**, даже без единого модуля.
- При запуске ядро **сканирует** `modules/` и автоматически подхватывает все найденные модули:
  добавляет их в интерфейс, показывает статус, позволяет запускать/останавливать.
- Пользователь может просто **добавить папку модуля** в `modules/` — при следующем запуске
  модуль появится в интерфейсе. Никакой перекомпиляции ядра не требуется.

## Стек

- **Бэкенд:** Go 1.25+, [Wails v2](https://wails.io/)
- **Фронтенд:** React 18 + Vite
- **Трей:** `fyne.io/systray`
- **Иконки/трей** взяты из оригинального ZPUI

## Сборка

Требования: Go 1.25+, Node.js 20+, [Wails CLI](https://wails.io/) v2.12+.

```bash
# фронтенд (один раз)
cd web && npm install

# сборка ядра
wails build -platform windows/amd64 -s -trimpath
```

Готовый `zpui.exe` появится в `build/bin/`. Рядом с ним создастся папка `modules/`.

## Разработка

```bash
wails dev    # hot-reload фронтенда + бэкенд
```

## Структура

```
main.go                      точка входа
internal/
  app/                       ядро Wails: lifecycle, API-биндинги, окно
  config/                    конфигурация (config.json)
  logger/                    ротируемый логгер
  singleinstance/            защита от повторного запуска
  executil/                  скрытый запуск процессов
  tray/                      системный трей
  modules/                   система модулей: discovery + менеджер процессов
web/                         React-фронтенд
build/windows/               иконки, манифест
modules/                     пользовательские модули (сканируется в рантайме)
```

Подробно о формате модулей — в [MODULES.md](MODULES.md).
