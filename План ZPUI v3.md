## 🧩 Шаг 3: Архитектура 6 модулей (детально)

Каждый модуль — **изолированный Go-пакет** с чётким интерфейсом. Прагматичный подход: не DLL (Go plugin не работает на Windows), а **сборка в один бинарник с строгим разделением пакетов через интерфейсы (dependency injection)**. Это даёт чистоту, тестируемость и простоту — то, что вы просите.

| # | Модуль | Назначение | Ключевые типы | Зависимости |
|---|---|---|---|---|
| **1** | `internal/zapret` | Управление `winws.exe`: установка/смена стратегий, winws-процесс, автотест через SSE | `Manager`, `Strategy`, `Process` | logger, storage, platform |
| **2** | `internal/proxy` | SOCKS5/HTTP прокси-сервер (наследник текущего `internal/proxy`), маршрутизация устройств | `Proxy`, `Device`, `Route` | logger, storage, monitor |
| **3** | `internal/monitor` | Снапшоты трафика (скорость DL/UL), детект устройств (arp/WMI), live-стрим | `Tracker`, `Snapshot`, `DeviceMonitor` | logger, storage |
| **4** | `internal/storage` | SQLite (WAL, миграции), репозитории, ротация | `DB`, миграции, репозитории | logger |
| **5** | `internal/logger` | slog + ротация файлов + PII-маскирование + аудит-логи в БД + контроль паник (recover) | `Logger`, `PiiMasker` | storage |
| **6** | `internal/platform` | Права админа, автозапуск, IPC с службой `zpuictl`, реестр Windows | `Admin`, `Autostart`, `ServiceClient` | logger |

**Поток данных:** `Wails (App.go)` → модули (через интерфейсы) → БД/сеть. Модули **не знают** про Wails — это чистый Go, можно тестировать без UI.

## 🔐 Шаг 4: Стратегия прав админа — Windows-служба `zpuictl`

Вы предложили 2 варианта. **Выбираю вариант 2 (служба)** — он правильнее архитектурно:

**Архитектура:**
- `service/main.go` → компилируется в `zpuictl.exe` (это уже частично есть в `development/zapretctl/`)
- Устанавливается как Windows-служба `ZPUIAdmin` через `golang.org/x/sys/windows/svc`
- IPC: **именованные каналы** (`\\.\pipe\zpuictl`) с JSON-протоколом команд
- Команды службы: `zapret.install`, `zapret.remove`, `zapret.strategy`, `winws.kill`, `windivert.clean`, `firewall.add`, `firewall.del`

**Приложение (ZPUI.exe)** работает **без прав админа**. Когда нужен админ-доступ:
1. UI вызывает `platform.ServiceClient.Execute("zapret.install", args)`
2. `ServiceClient` пишет запрос в pipe
3. Служба `zpuictl` (запущенная под SYSTEM) выполняет и возвращает результат
4. UI получает ответ

**В настройках** (вкладка «Настройки → Обновления и компоненты»):
- Кнопка «Установить службу» (требует одноразового UAC) → ставит `zpuictl.exe` как службу
- После установки — вкладка работает как обычно (без UAC при каждом действии)
- Кнопка «Удалить службу» (с подтверждением)
- Статус службы: запущена / остановлена / не установлена

Это решает вашу проблему авто-запуска без спама UAC.

## 🛠️ Wails API (биндинги Go → TypeScript)

Wails генерирует типобезопасные биндинги. В `app.go`:

```go
type App struct {
    ctx     context.Context
    zapret  zapret.Manager
    proxy   proxy.Manager
    monitor monitor.Tracker
    storage storage.DB
    logger  logger.Logger
    admin   platform.ServiceClient
}

// Методы, которые Wails exposes фронтенду:
func (a *App) ZapretStatus() (ZapretStatus, error)
func (a *App) ZapretStart(strategy string) error
func (a *App) ProxyStatus() (ProxyStatus, error)
func (a *App) MonitorSnapshot() (Snapshot, error)
func (a *App) DevicesList(filter string) ([]Device, error)
func (a *App) InstallService() error  // через UAC один раз
// ... и т.д.
```

Для live-данных (графики, статус) — **Wails Events** (`runtime.EventsEmit`) вместо SSE, что нативнее и быстрее.
