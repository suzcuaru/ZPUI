# Система модулей ZPUI

Модуль — это самостоятельная папка внутри `modules/`, расположенной рядом с `zpui.exe`.
Каждый модуль содержит свой собственный исполняемый файл (ядро) и может иметь любую
внутреннюю структуру и собственный жизненный цикл разработки/версионирования.

## Как это работает

1. Ядро ZPUI при запуске сканирует папку `modules/`.
2. Каждая подпапка с файлом `module.json` считается модулем.
3. Модули появляются в интерфейсе (раздел «Модули» + иконки в боковой панели).
4. Из интерфейса модуль можно запустить / остановить / перезапустить.
5. Чтобы добавить модуль — просто положите его папку в `modules/` и перезапустите ZPUI.

> Никакой перекомпиляции ядра не требуется. Ядро общается с модулем как с отдельным процессом.

## Структура папки модуля

```
modules/
└── my-module/
    ├── module.json        # манифест (обязательно)
    ├── my-module.exe      # ядро модуля (поле entry)
    ├── icon.svg           # иконка (опционально)
    ├── status.json        # модуль может писать сюда свой статус (опционально)
    └── ...                # любые доп. файлы модуля
```

## Манифест `module.json`

| Поле         | Тип       | Обяз. | Описание                                              |
|--------------|-----------|-------|------------------------------------------------------|
| `id`         | string    | да*   | Уникальный идентификатор (по умолчанию = имя папки)  |
| `name`       | string    | да*   | Отображаемое имя                                     |
| `version`    | string    | нет   | Версия модуля (семантика своя, напр. `1.0.0`)        |
| `author`     | string    | нет   | Автор                                                |
| `description`| string    | нет   | Описание                                             |
| `icon`       | string    | нет   | Имя файла иконки относительно папки модуля           |
| `entry`      | string    | да    | Имя .exe ядра относительно папки модуля              |
| `args`       | string[]  | нет   | Аргументы запуска                                    |
| `auto_start` | bool      | нет   | Запускать автоматически со стартом ZPUI             |
| `placements` | string[]  | нет   | Где показывать (`sidebar`)                           |
| `min_app`    | string    | нет   | Минимальная версия ядра ZPUI                         |

\* если не указаны, берётся имя папки.

### Пример

```json
{
  "id": "example-mod",
  "name": "Example Module",
  "version": "1.0.0",
  "author": "SuzucaRU",
  "description": "Демонстрационный модуль",
  "entry": "example-mod.exe",
  "args": [],
  "auto_start": false,
  "placements": ["sidebar"]
}
```

## Передача статуса (опционально)

Модуль может записывать файл `status.json` в свою папку с произвольными полями.
Ядро читает этот файл и отдаёт в интерфейс вместе с состоянием процесса.

```json
{
  "connections": 12,
  "uptime": "00:05:23",
  "custom_field": "любые данные"
}
```

## Контракт ядра модуля

Ядро ZPUI запускает `entry` как **скрытый** дочерний процесс (без окна).
Рабочая директория процесса = папка модуля. Модуль волен делать внутри своей папки
всё что угодно: сетевые операции, файлы, логи и т.д.

При остановке ядро завершает процесс (`Kill`). При завершении самого ZPUI —
останавливает все запущенные модули.

## Управление модулями

- **Запуск/Стоп/Рестарт** — из раздела «Модули».
- **Включить/Отключить** — отключённый модуль не запускается (даже с `auto_start`)
  и скрывается из автозапуска.
- Состояние процесса (running/stopped/error) опрашивается интерфейсом в реальном времени.

## Создание своего модуля

1. Создайте папку `modules/my-mod/`.
2. Положите туда свой `.exe` (на любом языке — Go, C#, Rust, C++ и т.д.).
3. Создайте `module.json` (см. пример выше).
4. Запустите/перезапустите ZPUI — модуль появится в интерфейсе.
